jQuery Dateinput @ jQuery Tools HTML5 date input for humans

HTML5 date input

HTML5 date input <input type="date" /> is currently supported by Opera and because it's a standard it will be implemented by other browsers as well. This brings HTML5 date picking for all browsers in consistent and standardized manner. The date field is just like any other FORM filed which can be submitted.

For nerds and humans

Call $(":date").dateinput(); and all your HTML5 date inputs are working. End users can pick dates with the mouse or with the keyboard arrow keys. Hackers can use VIM keyboard shortcuts to havigate between dates and months.

Styleable

You can design date pickers that fit to the overall feel of your site. You can use images and the latest CSS3 techniques if you like. CSS class names can be changed and TABLE's are not used so you can even design your own layout: be it single row or vertical. Professional example skins are provided and you can freely use them.

File size: 2.0 kb

And you'll have all the features and configuration options you'll possibly need such as localization, formatting, year/selectors, scripting API and an event model. Smaller codebase is easier to control and results to snappier behaviour.

Formatting

Formatting specifies how the dates are displayed for the user. Formatting pattern can be given in format configuration variable and for getDate() function in the API. Here are few examples:

$(".date").dateinput({ // this is displayed for the user format: 'You picked: dddd / dd.mm.yyyy', // another format is send to the server change: function() { var isoDate = this.getDate('yyyy-mm-dd'); $("#backendValue").val(isoDate); } });

Formatting characters can be mixed with other characters and you can have them as many as you like.

List of formatting characters

characters description

Skinning

Skinning is the same as writing CSS against the dateinput HTML layout. Here is an example CSS file and the default HTML layout. This tool allows you to customize the id and class names with the css configuration option. For example

$(".date").dateinput({ css: { prefix: null, root: 'dateinput', head: 'pickhead', } });

Prefix

All id's and class names are prefixed by the value specified in prefix property allowing you to have them on common "namespace". This avoids potential clashes with other CSS names on your web application. By setting this value to null the prefix is not used allowing you to use shorter names. Following table lists all properties supported by the css configuration option. By default the tool uses a prefix "pick" which has been colored in blue on the default value.

Events

First make sure you have read about Events in jQuery Tools. All event listeners receive Event Object as the first argument. Here is an example:

// select an input field and assign an event listener to it. $(":input.date").change(function(event, date) { console.info("a date was picked", date); });
event When does it occur?
onBeforeShow before dateinput is shown
onShow after the dateinput is completely shown (after a possible animation)
change when a date was picked and the field value has been changed. Note that the name is the same as the jQuery's change method and you can bind the event directly with that function. For other event's you must use the bind method. This leads to more fluent syntax as seen on the example above.
onHide when a dateinput is closed

Localizing

You can localize the dateinput to your language with a static $.tools.dateinput.localize call after which you can use the dateinput with another language. The tool uses following call internally to enable English localization.

// build-in localization $.tools.dateinput.localize("en", { months: 'January,February,March,April,May,June,July,August,September,October,November,December', shortMonths: 'Jan,Feb,Mar,Apr,May,Jun,Jul,Aug,Sep,Oct,Nov,Dec', days: 'Sunday,Monday,Tuesday,Wednesday,Thursday,Friday,Saturday', shortDays: 'Sun,Mon,Tue,Wed,Thu,Fri,Sat' });

To add French localization you do following.

// example French localization $.tools.dateinput.localize("fr", { months: 'Janvier,Février,Mars,Avril,Mai,Juin,Juillet,Août,' + 'Septembre,Octobre,Novembre,Décembre', shortMonths: 'Jan,Fév,Mar,Avr,Mai,Jun,Jul,Aoû,Sep,Oct,Nov,Déc', days: 'Dimanche,Lundi,Mardi,Mercredi,Jeudi,Vendredi,Samedi', shortDays: 'Dim,Lun,Mar,Mer,Jeu,Ven,Sam' });

Notice that all special characters need to be escaped. I used this list to find the special entities for French characters. After the localization is in place you can use French language by supplying lang: 'fr' on your configuration. Or you can make French language the default language by editing global configuration as follows: $.tools.dateinput.conf.lang = 'fr';
